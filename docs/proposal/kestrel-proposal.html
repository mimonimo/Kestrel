01서비스 개요

"무엇이 취약한가"를 넘어 "무엇부터 봐야 하는가"를 알려주는 취약점 인텔리전스 서비스.

매년 수만 건의 CVE 가 쏟아지지만 실제로 악용되는 비율은 일부입니다. Kestrel 은 흩어진 취약점 피드를 한곳에 모으고, 이론적 심각도(CVSS) · 악용 확률(EPSS) · 실제 악용 사실(KEV) 세 신호를 함께 보여 주어 보안 담당자가 우선순위를 빠르게 판단하도록 돕습니다.

제공 가치

  • 통합 수집 — 4개 공개 피드를 자동 수집·정규화하고 중복을 제거해 단일 코퍼스로 관리.
  • 우선순위 신호 — CISA KEV(실제 악용) · FIRST EPSS(악용 확률)로 CVSS 단독 판단의 맹점을 보완.
  • 빠른 검색·조회 — Meilisearch 오타 허용 검색과 심각도·소스·도메인 필터.
  • AI 분석 · 협업 — CVE 별 AI 분석, 북마크, 커뮤니티(게시글·댓글)로 팀 활용 지원.

06데이터 모델 & API

핵심 테이블은 본문과 3축 신호를 담는 vulnerabilities 이며, 그 위에 분석·커뮤니티·운영 테이블이 붙습니다.

6.1 핵심 스키마 (PostgreSQL)

-- 취약점 본문 + 우선순위 신호 (수집의 종착점)
CREATE TABLE vulnerabilities (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cve_id          VARCHAR(32) UNIQUE NOT NULL,
  title           VARCHAR(512) NOT NULL,
  description     TEXT NOT NULL,
  cvss_score      NUMERIC(3,1),                  -- 이론적 심각도 0.0–10.0
  severity        severity_enum,                 -- low|medium|high|critical
  epss_score      DOUBLE PRECISION,              -- 악용 확률 예측 (FIRST)
  kev_listed      BOOLEAN NOT NULL DEFAULT false,-- 실제 악용 사실 (CISA KEV)
  source          source_enum NOT NULL,          -- nvd|mitre|exploit_db|github_advisory
  raw_data        JSONB NOT NULL DEFAULT '{}',
  search_vector   TSVECTOR,                      -- 전문검색 폴백
  published_at    TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX ix_vuln_search   ON vulnerabilities USING GIN(search_vector);
CREATE INDEX ix_vuln_priority ON vulnerabilities (kev_listed, epss_score DESC, cvss_score DESC);

이 외 analysis_results(AI 분석), users·posts·comments(커뮤니티), bookmarks·tickets, login_logs·audit_logs(운영) 테이블이 함께 운영됩니다.

07운영 현황 & 향후 과제

현재 안정적으로 가동 중이며, 확장은 트래픽·필요에 맞춰 단계적으로 진행합니다.

7.1 운영 현황

수집 CVE ≈ 338,897건 KEV 등재 1,611건 출처 MITRE 93% · Exploit-DB · NVD · GHSA 단일 EC2 가동 일일 백업 SSM 전용 접속

  • 가용성 — Caddy + Docker Compose 자동 재시작, 데이터 볼륨 분리로 인스턴스 교체 시에도 데이터 보존.
  • 보안 — SSH 차단·SSM 접속, 비밀값은 부팅 시 무작위 생성, 접속·감사 로그 기록.
  • 데이터 보호 — AWS Backup 으로 데이터 볼륨 일일 스냅샷.

7.2 향후 과제 (단계적)

과제내용
자원 확장트래픽 증가 시 인스턴스 상향, 이후 DB/검색을 매니지드(RDS·OpenSearch 등)로 분리
도메인·인증kestrel.day 등 정식 도메인 적용 후 이메일 인증·OIDC 로그인 도입

관측 강화OpenTelemetry·Sentry 상시화로 수집·AI 파이프라인 가시성 확보

08결론

Kestrel 은 지금, 실제로 동작하는 취약점 인텔리전스 서비스다.

Kestrel 은 33만 건 이상의 CVE 를 수집해 CVSS·EPSS·KEV 3축으로 우선순위를 제시하고, 검색·대시보드·AI 분석·커뮤니티를 제공하는 운영 중인 웹 서비스입니다. 화려한 다중 클라우드 아키텍처 대신 단일 EC2 + Docker Compose + IaC 라는 단순하고 재현 가능한 구성을 택해, 적은 운영 비용으로 안정적으로 가동하고 있습니다.
